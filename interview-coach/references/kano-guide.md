# KANO模型应用指南 — 面试场景专用

## 核心概念

KANO模型将需求按"满足程度与用户满意度的关系"分为五类，面试中常用前三类：

### 必备型需求（Must-be）
- 做到了用户觉得理所当然，做不到用户愤怒
- 判断标准：负反馈中情绪烈度最高的需求
- 面试中的表达："这是用户的底线，不满足其他都免谈"
- 产品策略：最高优先级，第一阶段必须解决

### 期望型需求（One-dimensional）
- 满足程度和满意度成正比——做得越好越满意
- 判断标准：用户会主动对比竞品的维度
- 面试中的表达："这是用户会拿来比较的指标"
- 产品策略：持续优化，第二阶段重点

### 魅力型需求（Attractive）
- 用户之前没有预期，做了会惊喜
- 判断标准：竞品都没做、用户没有主动提过但行为暗示需要
- 面试中的表达："这是差异化机会，竞品还没做"
- 产品策略：建立差异化，第二/三阶段

## 面试中的注意事项

### 不要说的
- "我做了KANO问卷调查"（除非真的做了）
- "根据KANO模型的统计结果"（暗示有严格的定量分析）

### 可以说的
- "我用KANO模型的思维框架来做需求分层和优先级判断"
- "从负反馈数据看，排版问题符合必备型需求的特征——做不到用户愤怒"
- "追问功能用户之前没有预期，属于典型的魅力型需求"

### 被追问时的回答
Q："你做过KANO问卷吗？"
A："没有走标准的双向问卷流程。我是用KANO的思维框架来做判断——依据是负反馈数据和竞品对比。比如排版崩塌引发的情绪烈度远高于翻译不准，这符合必备型需求'做不到用户愤怒'的特征。"

## KANO与产品分期的对应关系

```
第一期 → 解决必备型需求 → 止血，做到及格
  ↓ 前提条件：必备型满足后才进入下一期
第二期 → 满足期望型 + 引入魅力型 → 提升满意度 + 建立差异化
  ↓
第三期 → 深化魅力型 / 智能化 → 扩大优势
```

关键逻辑：必备型需求没满足时做魅力型需求是无效的——用户连基础体验都没过关，不会有心情欣赏创新功能。
