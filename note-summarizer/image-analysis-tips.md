# 图片分析技巧参考

本文档供 Claude 在分析笔记图片时参考，涵盖各类图片场景的处理建议。

---

## 一、图片类型识别与分析策略

### 1. 截图（软件界面、网页）
**识别重点：**
- 标题栏、菜单、按钮文字
- 主要内容区域的文字
- 报错信息（优先级最高）
- URL 地址栏内容

**输出格式：**
```
[截图：{应用/网页名称} — {主要内容描述}。关键文字："{提取的重要文字}"]
```

### 2. 手写笔记
**识别重点：**
- 逐字识别，遇到不确定的字用 `[?]` 标注
- 保留原始标点和换行结构
- 注意箭头、下划线等强调符号

**输出格式：**
```
[手写："{识别文字}"]
```

### 3. 流程图 / 思维导图
**识别重点：**
- 节点文字
- 连线方向和标注
- 层级关系

**输出格式：**
```
[流程图：{根节点} → {分支1} / {分支2}；其中 {分支1} 包含 {子节点A}、{子节点B}]
```

### 4. 表格 / 数据图
**识别重点：**
- 表头
- 关键数值
- 趋势（上升/下降/稳定）
- 最大值、最小值、异常值

**输出格式：**
```
[表格：{标题}。列：{col1}、{col2}...。关键数据：{N}行，最高值 {X}，最低值 {Y}]
```

### 5. 公式 / 数学内容
**识别重点：**
- 用 LaTeX 或纯文字还原公式
- 说明公式的含义（若上下文可推断）

**输出格式：**
```
[公式：E = mc² （能量与质量关系）]
```

### 6. 图片/照片（实物、场景）
**识别重点：**
- 主体对象
- 与笔记主题的关联性
- 文字（若有）

**输出格式：**
```
[图片：{主体描述}，与笔记主题"{关联说明}"]
```

### 7. 代码截图
**识别重点：**
- 编程语言
- 函数/类名
- 关键逻辑
- 报错堆栈（若有）

**输出格式：**
```
[代码截图（{语言}）：{函数名/功能描述}。关键逻辑：{简述}]
```

---

## 二、base64 图片处理

当图片以 `data:image/...;base64,...` 格式出现时：
1. Claude 可直接在消息中处理 base64 图片（支持 PNG、JPG、WebP、GIF）
2. 将 base64 字符串作为 image content block 传递即可分析
3. 注意：大型 base64 图片可能占用较多上下文，优先分析关键图片

---

## 三、图片不可读时的处理

遇到以下情况时的降级策略：
- **图片文件不存在**：标注 `[图片缺失：{原始路径}]`，从 alt 文字推测内容
- **图片格式不支持**：标注 `[图片格式不支持：{格式}]`
- **图片过于模糊**：尽力识别，标注可信度低的部分
- **图片为纯装饰性**（如分割线）：直接跳过，不纳入总结

---

## 四、多图片笔记的处理顺序

1. 先完整读取全文，了解笔记主题
2. 按出现顺序逐一分析图片
3. 将图片分析结果标注在对应文字位置
4. 最终总结时，图片内容与文字内容自然融合，不单独列出

---

## 五、隐私与敏感内容

若图片包含以下内容，在总结中进行脱敏处理：
- 身份证/护照号码 → `[身份证号：已隐藏]`
- 银行卡号 → `[银行卡号：已隐藏]`
- 手机号 → `[手机号：1xx-xxxx-xxxx]`
- 密码/密钥 → `[敏感信息：已隐藏]`
