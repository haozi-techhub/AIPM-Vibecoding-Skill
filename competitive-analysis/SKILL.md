# 竞品分析工作流 Skill

产品经理在评估新产品方向时，按 **7步顺序** 完成从"发现机会"到"立项报告"的完整调研链路。每步必须用 browse 工具从真实网页获取数据，禁止杜撰，无法获取的内容明确告知用户。

---

## 使用前置条件

用户需提供：
1. **产品方向**：具体产品名称/功能描述
2. **目标用户假设**：3~5类用户群体
3. **核心场景假设**：用户在什么情况下使用
4. **成功标准**：什么情况下"值得做"（如市场增速、竞争格局、未满足痛点）

---

## Step 1：决策问题拆解

**触发词**：用户提供产品方向和成功标准后，立即执行

拆解出"做/不做"决策所需回答的所有子问题清单：

```
分析目标：{产品方向}
决策类型：做/不做

需要回答的子问题：
□ 市场层：规模多大？增速多少？生命周期？
□ 竞争层：直接竞品是谁？护城河高低？
□ 用户层：谁在用？为什么用？痛点是什么？
□ 功能层：竞品做了什么？没做什么？
□ 商业层：怎么赚钱？成本结构？
□ 差异化：我们能做别人做不了的是什么？
```

输出：结构化问题清单，确认用户对齐后进入Step 2。

---

## Step 2：竞品识别

**Browse 搜索渠道**（并行执行，提升效率）：

```
搜索平台：
- 36氪：搜索"{产品方向} 融资" "{产品方向} 创业"
- ProductHunt：搜索英文关键词
- 小红书：搜索"{产品方向}" "{核心场景}"
- IT桔子：搜索相关公司融资记录
- App Store：搜索相关App，查看下载量和评分
- 垂直社区：行业官网/论坛（根据产品方向补充）
```

**竞品分类输出**：

| 类别 | 定义 | 要求输出字段 |
|------|------|------------|
| 直接竞品 | 做一样的事，抢一样的用户 | 名称/官网/核心功能/目标用户/融资阶段/信息来源 |
| 间接竞品 | 解决类似需求但方式不同 | 同上 |
| 替代品 | 用户现在实际用什么解决 | 同上 |
| 潜在竞品 | 有能力进来但还没进的大公司 | 同上 |

**规则**：每条信息必须标注来源URL，无法获取的字段注明"待补充"，禁止推测。

## Step 3：市场层分析

**Browse 搜索渠道**：
- 艾瑞咨询/易观/CBNData：搜索行业报告
- 36氪/虎嗅：搜索融资新闻和市场分析
- 政府/协会官网：行业数据

**必须输出**（每个数据标注来源+年份）：

```markdown
## 市场规模
- 中国市场：___亿元（来源：___，___年）
- 全球市场：___亿美元（来源：___，___年）
- 预测规模：___亿元（___年，来源：___）

## 增长趋势
- CAGR：___%
- 生命周期阶段：□导入期 □成长期 □成熟期 □衰退期
- 关键增长驱动因素：

## 竞争格局
- CR5市场份额：___%
- 最近一年大额融资（>1000万）：
- 大公司进场情况：
```

## Step 4：竞品商业模式对比

**Browse 任务**：访问每个竞品的官网定价页 + App Store 内购页

**输出格式**：

```markdown
## {竞品名} 商业模式

### 收费方式
| 套餐 | 价格 | 功能 |
|------|------|------|
| 免费版 | $0 | ... |
| 付费版 | $/月 | ... |

### 算力成本估算
生成一张图的成本：¥___（来源/推算方式）

### 壁垒分析
- 技术壁垒：高/中/低（说明）
- 数据壁垒：高/中/低（说明）
- 品牌壁垒：高/中/低（说明）
```

**规则**：拿不到的数据明确说明"无法获取，建议用户实测"，禁止估算当做事实。

## Step 5：用户层分析

**Browse 搜索渠道**（必须全部搜索）：
- 小红书：搜索"{产品} 体验" "{产品} 推荐" "{核心场景}"
- 知乎：搜索"{产品} 哪个好" "{产品} 对比"
- App Store：抓取竞品的1-3星差评和5星好评原文
- Reddit：搜索英文关键词（如有海外竞品）

**输出框架**：

## 执行规则（所有步骤通用）

1. **数据必须标来源**：每个数据点附URL或报告名称+年份
2. **禁止杜撰**：拿不到的内容明确说"无法获取，请用户补充"
3. **信息有冲突时**：列出多个来源，说明差异，不擅自取舍
4. **Browse工具策略**：优先直接抓取官网/App Store，次选搜索引擎，两者都失败才用用户补充
5. **步骤可并行**：同一步骤内的多个竞品可并行抓取，提升效率
6. **发现重要信息立即标注**：如"阿里API文档明确不支持汉服"这类关键发现要突出显示
