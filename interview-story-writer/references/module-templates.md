# 各模块文档产出模板参考

> 根据候选人的角色和需求选择需要的模块，按以下模板产出

---

## 用户场景痛点分析模板

每个场景按以下链条展开：
```
场景还原（用户在干嘛）
  → 遇到什么问题（痛点）
  → 试过什么方案（现有解决方案）
  → 为什么没解决（现有方案的缺陷）
  → 我们怎么解决的（产品的解法）
  → 本质是什么（一句话总结这个场景的核心价值）
```

---

## Agent架构设计文档模板

建议结构：
1. 为什么需要多Agent（单Agent方案的瓶颈）
2. 架构迭代历程（版本表格：v0.1→v0.5→v1.0每次迭代的问题和决策）
3. 最终架构总览（ASCII架构图）
4. 架构设计模式选择及理由
5. 各Agent详细设计（角色定义+prompt要点+设计理由）
6. 全局上下文设计（数据结构+读写权限矩阵）
7. 架构迭代故事（面试必讲的踩坑→解决叙事）
8. 面试Q&A

---

## 知识库设计文档模板

每个知识库按以下结构：
1. 给谁用的（服务哪个Agent）
2. 存什么（用具体例子说明，如孙悟空的"角色身份证"）
3. 数据怎么写进去的（来源+流程+质量把控）
4. 怎么被检索到的（检索策略）
5. 查不到时怎么办（降级方案）

额外章节：
- 知识库之间的协作关系（一个端到端案例）
- 角色一致性/数据一致性怎么通过知识库保障
- 管理总览（谁负责/更新频率/入库流程/版本管理）

---

## 评测体系文档模板

建议结构：
1. 为什么需要评测体系
2. 评测范围（端到端各环节）
3. 评测维度与指标体系（维度表格+每个维度的具体指标）
4. 三层评测方法论（每层怎么做/适合评什么/不适合评什么）
5. 评测流程（什么时候评/评完怎么用）
6. 上线红线
7. 评测驱动迭代闭环
8. 面试Q&A

---

## 评测集文档模板

建议结构：
1. 评测集是什么/为什么需要
2. 整体设计（规模/子集划分）
3. 场景分布设计（为什么和线上不一致）
4. 各子集详细设计（每个都给"一条数据长什么样"的完整示例）
5. 怎么构建的（种子集→内测扩充→线上badcase补充）
6. 质量标准
7. 维护机制（更新/版本管理/保鲜策略）
8. 面试Q&A

---

## 产品架构文档模板

三层递进：
1. 宏观：产品在整个App/公司中的位置和战略角色
2. 中观：功能模块全景+用户路径+功能优先级矩阵
3. 微观：技术产品分层架构+核心数据流

额外章节：
- 关键架构决策与设计取舍
- 架构如何支撑业务指标
- 架构演进路径（面试可讲的规划视角）

---

## 简历项目描述模板

格式：
```
项目名 · 产品名 | 角色 | 时间段

一句话概括（产品是什么+你负责什么）

模块1标题：动作+方法+量化成果
模块2标题：动作+方法+量化成果
模块3标题：动作+方法+量化成果
模块4标题：动作+方法+量化成果
```

要求：
- 每条都带量化数据
- 包含岗位关键词（根据目标岗位调整）
- 准备3个版本：完整版/标准版/精简版

---

## 追问题库模板

分类：
- 架构设计类
- 技术方案类（Prompt/RAG/知识库等）
- 用户和产品类
- 评测类
- 协作和软技能类
- 方法论和认知类
- 防守类（答不上来时的兜底话术）

每道题格式：
```
追问概率（⭐⭐⭐/⭐⭐/⭐） + 问题 + 推荐回答要点
```
