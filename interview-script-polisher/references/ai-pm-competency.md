# AI产品经理核心能力模型 — 面试场景专用

## 使用说明

本文档定义AI PM面试的核心能力维度。每个维度标注：能力含义、面试中如何展示、可用的表达模板、常见失误。在准备面试故事和优化面试稿时，对照本模型确保覆盖关键能力。

---

## 核心能力全景

```
用户研究 → 产品定义 → Agent/模型架构 → Prompt工程 → 评测体系 → 数据/指标 → 干系人管理
   ↑                                                                          |
   └──────────────── 迭代闭环：结果验证 → 洞察 → 调整 ←─────────────────────────┘
```

AI PM的能力链条是一个闭环：从理解用户出发，经过产品设计和技术选型，通过评测和数据验证，再回到用户洞察。面试中要展示的不是每个单点能力，而是这个闭环运转的完整性。

---

## 能力维度详解

### 1. 用户研究

**能力含义**：理解用户的真实需求和使用动机，区分"用户说的"和"用户要的"，将洞察转化为产品决策依据。

**面试中如何展示**：
- 讲清调研方法和调研过程（不只是结论）
- 展示从数据/行为中发现反直觉洞察的能力
- 用户分层要与后续产品策略对应

**可用表达模板**：
- "我在分析用户反馈时发现了一个反直觉的现象——"
- "用户嘴上说要XX，但行为数据告诉我们真正的痛点是YY"
- "我按使用动机把用户分成三类，每类对应不同的产品策略"

**常见失误**：
- 调研停留在"问了几个用户" → 样本不够、方法不系统
- 分层完不引用 → 说明分层是装饰性的
- 把用户原话当结论 → 缺乏洞察提炼能力

---

### 2. 产品定义

**能力含义**：将模糊的需求转化为清晰的产品方案。包括：核心价值主张、MVP范围、功能优先级、产品节奏。

**面试中如何展示**：
- 一句话说清产品是什么、解决什么问题
- 展示MVP取舍的逻辑（不是什么都做，要有克制）
- 功能优先级有明确的判断框架

**可用表达模板**：
- "一句话说就是：帮XX用户解决YY问题，通过ZZ方式"
- "一期我做了一个克制的决定——只做三个核心功能"
- "优先级判断的框架是：先必备型需求止血，再期望型需求提升"

**常见失误**：
- 产品定义说了三分钟还没讲清楚 → 定义能力弱
- MVP什么都想要 → 缺乏优先级判断
- 功能列表没有逻辑 → 是堆砌不是设计

---

### 3. Agent/模型架构

**能力含义**：理解AI技术方案的选型逻辑，能评估不同模型/架构的优劣势，知道技术选型怎么影响产品体验。

**面试中如何展示**：
- 讲清为什么选这个模型/架构（对比过哪些方案）
- 知道模型能力边界在哪，超出边界怎么处理
- 展示"产品需求 → 技术选型"的推导逻辑

**可用表达模板**：
- "我们对比了三个方案，最终选XX，因为在我们的场景下它在YY维度明显更优"
- "这个模型的能力边界是XX，超出这个范围我们用YY方式兜底"
- "架构上我推动了从A切到B，关键依据是评测数据"

**常见失误**：
- 只说"用了GPT-4"但不知道为什么 → 缺乏技术判断力
- 不知道模型能力边界 → 产品设计有盲区
- 技术选型和产品需求脱节 → 为了用技术而用技术

---

### 4. Prompt工程

**能力含义**：将产品逻辑转化为Prompt设计，系统化管理和迭代Prompt，理解Prompt对产品体验的直接影响。

**面试中如何展示**：
- 讲Prompt设计的方法论（不只是"我写了一段Prompt"）
- 展示Prompt迭代的过程和效果量化
- 说明Prompt管理的机制（版本控制、AB测试）

**可用表达模板**：
- "Prompt在我们产品里本质上就是产品逻辑——我把XX规则编码进Prompt里"
- "Prompt迭代不是随便改改，我建立了评测-修改-验证的闭环"
- "通过Prompt优化，XX指标从A提升到B"

**常见失误**：
- 把Prompt当成一次性的事 → 缺乏迭代意识
- 说不清Prompt改了什么、效果怎样 → 缺乏量化思维
- 不理解Prompt和模型能力的关系 → Prompt再好也突破不了模型上限

---

### 5. 评测体系

**能力含义**：设计系统化的AI效果评估方案，包括评测维度、评测集、评测流程、评测结果的应用。这是AI PM区别于传统PM的核心能力。

**面试中如何展示**：
- 讲清评测维度是怎么定义的（不是拍脑袋）
- 展示评测集的构建方法（来源、规模、质量控制）
- 说明评测结果怎么驱动产品决策

**可用表达模板**：
- "我定义了四个评测维度，依据是用户最关注的体验因素"
- "评测集用的是线上真实case，500条，覆盖了主要场景和边界情况"
- "评测结果出来：四个维度里三个赢一个平，数据摆出来讨论空间就很小了"

**常见失误**：
- 没有评测体系 → AI PM核心能力缺失
- 评测只有"让人看看" → 不够系统化
- 评测维度和用户体验脱节 → 评了但没评到点上
- 评测集太小或质量差 → 结论不可靠

---

### 6. 数据/指标

**能力含义**：设计和运用数据体系衡量产品效果，包括指标定义、埋点设计、数据分析、数据驱动决策。

**面试中如何展示**：
- 讲清北极星指标的选择逻辑
- 展示指标体系的层次结构（指标树）
- 有"从数据中发现问题并解决"的具体案例

**可用表达模板**：
- "北极星指标选的是XX，因为它最能代表用户获得的核心价值"
- "这个指标我拆成三层：结果指标→过程指标→行为指标"
- "有一个指标的异常波动让我注意到了问题——深入分析后发现是XX原因"

**常见失误**：
- 只有结果指标没有过程指标 → 数据体系不完整
- 指标之间没有逻辑关系 → 拼凑的不是设计的
- 说不出具体数字 → 可信度下降
- 数据只用来汇报不用来决策 → 没有数据驱动的实质

---

### 7. 干系人管理

**能力含义**：协调技术、设计、运营等多方资源推动产品落地，化解分歧、管理预期、高效沟通。

**面试中如何展示**：
- 讲具体的冲突/分歧场景和解决方式
- 展示"不同干系人用不同沟通策略"的意识
- 体现推动力但不是强推

**可用表达模板**：
- "推动这件事不容易——等于推翻现有方案，阻力是有的"
- "我的策略是不讲理论，用数据说话，让结论变得显而易见"
- "我没有试图说服他们，而是帮他们消除最大的顾虑"

**常见失误**：
- 把冲突归因为"别人不配合" → 缺乏责任感
- 只说"我沟通了" → 太泛，需要具体方法
- 强调自己的英雄主义 → 团队协作能力存疑

## 能力与STAR叙事的连接

### 将能力融入故事而非单独罗列

面试中不要说"我具备评测体系设计能力"，而是在故事中自然展示这个能力。

**连接方法**：
- Situation/Task → 展示用户研究、产品定义
- Action → 展示架构选型、Prompt工程、评测体系、数据设计
- Result → 展示数据/指标能力、迭代闭环

**示例**：
```
"我发现用户最大的痛点不是翻译不准，而是排版崩溃"  → 用户研究能力
"一期我只做三个核心功能，追问功能放到二期"        → 产品定义能力
"对比了三个方案，用评测数据推动了架构切换"         → 架构选型 + 评测能力
"通过Prompt优化，准确率从78%提升到88%"           → Prompt工程能力
"北极星指标选的是满意度评分，拆成三层指标树"       → 数据体系能力
"推动架构切换时，我用数据说话而不是讲理论"         → 干系人管理能力
```

## AI PM与传统PM的核心差异

### 传统PM做不到、AI PM必须做的事

| 差异维度 | 传统PM | AI PM |
|---------|--------|-------|
| 迭代依据 | 用户反馈 + AB测试 | 评测体系驱动，模型评测是核心迭代手段 |
| 能力边界 | 功能做了就有 | 模型有能力上限，产品设计必须考虑边界 |
| 产品逻辑 | 写在PRD和代码里 | Prompt就是产品逻辑，需要系统管理 |
| 质量控制 | 测试用例覆盖 | 评测集 + 线上监控 + 分布健康度 |
| 效果衡量 | 功能使用率/转化率 | 准确率/满意度/模型效果指标 |
| 技术理解 | 了解技术可行性 | 理解模型原理和能力边界，参与技术选型 |

### 面试中如何体现差异化

关键信号词（面试官听到这些会加分）：
- "评测驱动迭代" — 不是拍脑袋优化，而是有系统评测
- "模型能力边界" — 知道AI能做什么不能做什么
- "Prompt即产品逻辑" — 理解AI产品的独特性
- "数据分布健康度" — 不只看平均值，关注分布
- "评测集构建" — 知道好数据是好评测的前提
- "降级方案" — AI不是万能的，需要兜底

### 面试中如何表达

```
❌ "我和传统PM不一样，我懂AI技术"（空泛，且可能引起反感）

✅ "在这个项目里，我发现AI产品的关键不是'功能做了就行'，而是
   要持续评估模型效果。所以我建立了评测体系，每次Prompt迭代
   都跑一遍评测集，确保改进是可量化的。这是AI产品和传统产品
   最大的区别——你不是做完就结束了，而是需要持续跟踪模型表现。"
```
