# 产品全流程面试清单

## 使用说明

面试中讲项目时，面试官会从以下环节追问。本清单帮助你确保每个环节都准备到位。每个环节列出：面试要展示什么、关键问题、追问信号、常见踩坑。

---

## 全流程环节

### 1. 需求背景与市场判断

**面试要展示**：为什么做这个产品/功能？市场窗口期在哪？你怎么判断"该做"而不是"可以做"？

**关键问题**：
- 行业趋势是什么？你的判断依据是什么（数据/事件/技术拐点）？
- 竞争格局怎样？为什么现在做而不是更早或更晚？
- 公司战略定位是什么？这个产品怎么服务于公司战略？

**追问信号**：
- 面试官问"为什么是这个时间点做" → 想听市场窗口判断
- 面试官问"公司为什么做这个方向" → 想听战略理解力
- 面试官问"你们有什么优势" → 想听资源禀赋分析

**常见踩坑**：
- 只说"老板让做的" → 没有展示自己的判断力
- 说了大量行业数据但没有结论 → 分析能力存疑
- 市场判断和后续产品决策脱节 → 说明背景是装饰性的

---

### 2. 用户研究与分层

**面试要展示**：用户是谁？怎么做的调研？分层依据是什么？分层结果如何指导产品决策？

**关键问题**：
- 用调研方法（定量/定性/数据分析）？样本量和来源？
- 按什么维度分层？为什么选这个维度而不是别的？
- 不同用户类型的核心差异是什么？产品策略有何不同？

**追问信号**：
- 面试官问"你怎么知道用户需要这个" → 想听调研过程和洞察
- 面试官问"用户为什么用你的产品" → 想听动机理解
- 面试官问"不同用户有什么区别" → 想听分层质量

**常见踩坑**：
- 分层只是贴标签，后面没有引用 → 分层变成装饰
- 分层维度是人口统计学（年龄/性别）而非使用动机 → 缺乏决策价值
- 调研结论和产品设计之间逻辑断裂 → 说明调研是事后补的

---

### 3. 竞品分析与差异化

**面试要展示**：你了解竞争对手吗？你的产品凭什么赢？差异化是真差异还是自我安慰？

**关键问题**：
- 核心竞品是谁？你怎么选的竞品范围（直接竞品/间接竞品/替代方案）？
- 竞品的优劣势是什么？你的判断来自实际体验还是二手信息？
- 你的差异化点是什么？能持续多久？壁垒在哪？

**追问信号**：
- 面试官问"XX产品也在做这个" → 想看你是否了解竞品动态
- 面试官问"用户为什么不用竞品" → 想听真实差异化
- 面试官问"竞品跟进怎么办" → 想听壁垒思考

**常见踩坑**：
- 说"我们没有竞品" → 不可信，替代方案也是竞争
- 只做功能对比表 → 没有产品判断
- 差异化点经不起追问（如"我们体验更好"但没有数据支撑） → 空泛

---

### 4. 产品定义与架构设计

**面试要展示**：产品的核心价值主张是什么？整体架构怎么支撑业务目标？

**关键问题**：
- 产品的一句话定义是什么？核心价值主张（value proposition）是什么？
- 功能模块怎么划分？模块之间什么关系？
- MVP包含什么？为什么是这些功能而不是别的？

**追问信号**：
- 面试官问"能不能一句话说清楚这个产品" → 考察产品定义能力
- 面试官问"第一版做了哪些功能" → 想听MVP取舍逻辑
- 面试官问"为什么不先做XX" → 考察优先级判断

**常见踩坑**：
- 产品定义模糊，说了五分钟还没讲清楚做什么 → 定义能力弱
- MVP功能太多，没有取舍 → 缺乏产品节奏感
- 架构设计和业务目标脱节 → 为了设计而设计

---

### 5. 交互设计与核心流程

**面试要展示**：核心用户流程是什么？关键交互节点怎么设计的？为什么这样设计？

**关键问题**：
- 用户从进入到完成核心任务的完整路径是什么？
- 关键交互节点（首次体验/核心操作/异常处理）怎么设计？
- 设计决策背后的依据是什么（数据/用研/最佳实践）？

**追问信号**：
- 面试官问"用户怎么完成XX操作" → 想听具体流程
- 面试官问"这里为什么这样设计" → 想听设计决策依据
- 面试官问"用户遇到错误怎么办" → 考察异常场景覆盖

**常见踩坑**：
- 只讲happy path，不讲异常处理 → 考虑不周全
- 交互设计没有数据支撑 → "我觉得"不够
- 流程描述太细碎，抓不住重点 → 面试时间管理差

---

### 6. AI能力设计（模型/评测/Prompt）

**面试要展示**：你怎么设计AI能力？怎么评估AI效果？怎么迭代优化？

**关键问题**：
- 选了什么模型/方案？为什么选这个？对比过哪些选项？
- 评测体系怎么设计的？评测维度和指标是什么？
- Prompt怎么设计和迭代的？有没有系统化的方法？
- 模型能力边界在哪？超出边界怎么处理？

**追问信号**：
- 面试官问"你们用的什么模型" → 入口问题，后面会深挖
- 面试官问"怎么评估效果" → 核心考察点，想听评测体系
- 面试官问"准确率多少" → 想听你对评测指标的理解深度
- 面试官问"模型不好怎么办" → 想听迭代方法论

**常见踩坑**：
- 只说"用了GPT-4"但说不清为什么 → 缺乏技术判断力
- 没有评测体系 → AI PM最核心的能力缺失
- 把评测等同于"让人看看效果" → 不够系统化
- 不知道模型能力边界 → 产品设计会有盲区

---

### 7. 数据体系（指标/埋点）

**面试要展示**：你用什么数据衡量产品成功？埋点体系怎么设计？数据怎么驱动决策？

**关键问题**：
- 核心指标（北极星指标）是什么？为什么选这个指标？
- 指标体系怎么搭建的？指标之间什么关系（指标树）？
- 埋点方案怎么设计？覆盖哪些关键行为？
- 有没有通过数据发现问题并驱动迭代的案例？

**追问信号**：
- 面试官问"你怎么衡量成功" → 想听北极星指标选择逻辑
- 面试官问"数据表现怎么样" → 想听具体数字和分析
- 面试官问"怎么发现的这个问题" → 想听数据驱动决策的过程

**常见踩坑**：
- 只有结果数据没有过程数据 → 数据体系不完整
- 指标之间没有逻辑关系 → 是凑的不是设计的
- 说不出具体数字 → 可信度打折
- 数据分析停在"看报表" → 没有主动洞察

---

### 8. 结果验证与迭代

**面试要展示**：产品上线后效果如何？你怎么验证假设？怎么基于结果迭代？

**关键问题**：
- 上线后核心指标变化如何？达到预期了吗？
- 如果没达预期，原因是什么？做了什么调整？
- AB测试怎么设计的？实验组/对照组怎么分？
- 迭代的优先级怎么排？依据是什么？

**追问信号**：
- 面试官问"效果怎么样" → 想听具体数据和归因
- 面试官问"如果重新做会怎么改" → 考察反思能力
- 面试官问"怎么知道是你的功能带来的提升" → 考察归因能力

**常见踩坑**：
- 只说"数据提升了XX%"但说不清归因 → 可能是其他因素
- 没有失败案例 → 不真实
- 迭代策略和数据分析脱节 → 迭代是随机的不是数据驱动的

---

### 9. 项目管理与跨团队协作

**面试要展示**：你怎么推动项目落地？怎么协调各方资源？遇到阻力怎么解决？

**关键问题**：
- 项目涉及哪些团队？你的角色和职责是什么？
- 遇到过什么推进阻力？怎么解决的？
- 项目节奏怎么把控？有没有delay？怎么处理的？
- 怎么做干系人管理？上级/平级/下游怎么沟通？

**追问信号**：
- 面试官问"团队怎么配合的" → 想听协作模式
- 面试官问"遇到过什么困难" → 想听解决问题的能力
- 面试官问"技术说做不了怎么办" → 考察沟通和推动力

**常见踩坑**：
- 只讲自己做了什么，不讲团队协作 → 显得独断
- 把推进阻力归因为别人 → 缺乏责任感
- 没有具体的冲突解决案例 → 故事不够丰满

---

## 自检方法

### 快速自检（讲项目前过一遍）

对照以上9个环节，问自己：
1. 每个环节我能讲1-2分钟吗？
2. 每个环节我有具体的数据/案例支撑吗？
3. 关键决策点我能说清"为什么"吗？
4. 如果面试官从任何一个环节追问，我能往下讲3个层次吗？

### 薄弱环节识别

如果某个环节只能讲出结论但说不清过程 → 这是薄弱环节，需要提前准备追问预案。

### 环节之间的逻辑链

确保环节之间有因果关系：
```
市场判断 → 决定了产品方向
用户分层 → 决定了优先级排序
竞品分析 → 决定了差异化策略
产品定义 → 承接以上分析的结论
AI能力设计 → 服务于产品定义
数据体系 → 验证产品假设
结果验证 → 闭环回到需求判断
```
