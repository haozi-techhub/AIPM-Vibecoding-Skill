# 用户分层方法论 — 面试场景专用

## 核心原则：按使用动机分层，不按人口统计学分层

### 为什么不按年龄/性别/职业分？

人口统计学分层（学生/白领/宝妈）描述的是"用户是谁"，但不能直接指导产品决策。同样是25岁的学生，翻论文和翻菜单的需求完全不同。

按使用动机分层描述的是"用户要解决什么问题"，直接对应产品需要提供什么能力。

### 分层方法

**Step 1：列举所有使用场景**
从数据和访谈中穷举用户的使用场景。

**Step 2：按底层动机归类**
问："用户做这件事，最终要解决什么问题？"将场景按最终目的归类。

**Step 3：命名用户类型**
用动机命名，不用身份命名。

| ❌ 身份命名 | ✅ 动机命名 | 为什么更好 |
|-----------|-----------|-----------|
| 学生用户 | 学习型用户 | 不是所有学生都在学习，职场人也可能在学习 |
| 出境游旅客 | 行动型用户 | 核心是"做决策"，海淘用户也有同样动机 |
| 职场人 | 效率型用户 | 核心是"结果可直接使用" |

**Step 4：为每类用户标注四个维度**

| 维度 | 说明 |
|------|------|
| 典型场景 | 这类用户最常见的使用场景 |
| 底层动机 | 使用产品要解决的"最终问题" |
| 核心期望 | 对产品最重要的要求 |
| 容忍底线 | 什么问题绝对不能出 |

**Step 5：验证分层的产品决策价值**

检验标准：每一类用户是否能对应到不同的产品策略？如果所有用户类型对应的策略是一样的，说明分层粒度不够或分层维度选错了。

## 分层与产品规划的对应

分完层后，必须在后续叙事中引用：

```
第一期 → 主要服务哪类用户？为什么先服务他们？
第二期 → 扩展到哪类用户？什么功能主要服务他们？
关键功能 → 这个功能主要满足哪类用户的什么动机？
```

如果分层做完后面没有引用，说明分层只是"装饰"，需要重新审视。

## 面试中的注意事项

不要说："我的用户是学生、旅客和职场人"（身份标签，没有决策价值）。

要说："我按使用动机把用户分成三类——学习型、行动型和效率型。他们用同一个功能，但要解决的最终问题完全不同，所以我们的产品策略对这三类也不同。比如一期优先解决效率型用户的排版痛点，因为他们占比最大且排版是硬需求；二期追问功能主要服务行动型用户，因为他们的核心动机是'翻完做决策'。"
